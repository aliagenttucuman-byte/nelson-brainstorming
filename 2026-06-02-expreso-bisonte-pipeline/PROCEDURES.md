# PROCEDURES — Biblioteca de lógica reusable

## Rol de procedures
Convertir conocimiento tácito del usuario en activos reutilizables.

## Estructura de un procedimiento
- Nombre
- Objetivo
- Resultado esperado
- Modelo recomendado
- Frecuencia de uso (`run_count`)
- Prioridad (`sort_order`)

## Ciclo de vida
1. Nace desde una ejecución exitosa.
2. Se reutiliza con botón "Usar".
3. Se prioriza por orden manual.
4. Se duplica para variantes.
5. Se elimina cuando queda obsoleto.

## Buenas prácticas
- Nombres orientados a negocio (no técnicos).
- Un procedimiento por necesidad concreta.
- Revisar y depurar biblioteca periódicamente.

## Próximas mejoras
- Tags por dominio (finanzas, operaciones, logística).
- Recomendación automática de procedimiento por prompt parecido.
- Métrica de efectividad por procedimiento.
