# Unsloth vs Oumi — Spike Comparativo para Alegent ForestAI

## Resumen Ejecutivo
Este brainstorming define un spike corto para comparar **Unsloth** y **Oumi** usando un caso representativo de Alegent ForestAI.

Objetivo: decidir rápido y con evidencia cuál usar para:
1) experimentación/fine-tuning rápido,
2) estandarización de pipeline end-to-end.

## Hipótesis (formato Nelson)
CREEMOS QUE ejecutar un spike comparativo Unsloth vs Oumi sobre un caso real de Alegent ForestAI,
RESULTARÁ en una decisión Go/No-Go por herramienta y una estrategia combinada clara,
CRITERIOS DE ACEPTACION:
- obtener primer resultado funcional en menos de 48h,
- medir complejidad de integración real,
- definir recomendación final con matriz objetiva.

## Pregunta de decisión
- ¿Conviene adoptar Unsloth, Oumi o enfoque híbrido (Unsloth para iteración + Oumi para estandarización)?

## Entregables
- `PLAN.md` — plan de ejecución del spike.
- `MATRIZ-EVALUACION.md` — criterios y scoring comparativo.
- `DECISION.md` — conclusión final (GO / NO-GO / HÍBRIDO).

## Criterio de éxito
